Guia Senior - Roadmap Dofus 2.10 .NET

Fecha: 2026-03-23

Proyecto: Dofus 2.10 Private Server

Objetivo: seguir un plan profesional, escalable y jugable por vertical slices.

1. Vision Del Proyecto

Construir un emulador Dofus 2.10 en .NET con arquitectura limpia, trazabilidad de protocolo y una base de datos controlada por nosotros.

La meta no es solo conectar el cliente, sino dejar una base que soporte autenticacion, personajes, mundo, inventario, NPCs y combate sin deuda tecnica innecesaria.

2. Estado Actual

Completado
  • AuthServer funcional en .NET
  • GameServer funcional en .NET
  • Login completo desde cliente
  • Seleccion de servidor operativa
  • Ticket auth-to-game operativo
  • CharactersList vacia respondida por el host
Hallazgo Clave

El bug principal de desconexion venia del bootstrap capturado. Se estaba enviando una cola falsa 1/1, no un problema real del cliente ni del socket base.

3. Principios De Diseno

  1. La DB de runtime es nuestra y no depende de SQLs antiguas de terceros.
  2. El contenido estatico debe importarse desde el cliente siempre que sea posible.
  3. El protocolo y la logica de negocio no deben mezclarse.
  4. Cada fase debe cerrar un flujo real del cliente.
  5. El proyecto debe crecer por vertical slices, no por bloques aislados.

4. Arquitectura Objetivo

Capa Host

Listeners, sockets, protocolo auth/game, sessions, tickets, logging de trafico y composicion del host.

Capa Bll

Casos de uso, reglas de negocio, cuentas, personajes, mundo y servicios de aplicacion.

Capa Data

EF Core, AppDataContext, UnitOfWork, repositories, persistencia, seeds e importers.

Capa Helper

Utilidades puras, extensiones, conversores y piezas compartidas sin dominio.

5. Estrategia Senior Para La Base De Datos

La recomendacion es no depender de una base SQL antigua encontrada en internet como fuente principal del emulador.

Decision recomendada: base propia desde cero para runtime y contenido estatico importado desde el cliente.

Por que no usar una SQL antigua como base oficial

  • Puede no coincidir con tu cliente 2.10 real.
  • Suele traer custom de otro servidor.
  • Introduce ids y estructuras que no controlamos.
  • Te deja acoplado a un modelo historico y opaco.

6. Modelo De Datos Recomendado

  • SpellTemplates

7. Estrategia De Contenido

No conviene crear a mano razas, mapas y contenido masivo. La ruta buena es construir importadores y poblar solo el primer vertical slice jugable.

Origen Uso Objetivo
Cliente 2.10 Extraer data estatica Razas, textos, mapas y metadatos
DB propia Persistencia runtime Cuentas, personajes, posiciones, tickets
Tools/Importers Transformacion controlada Evitar carga manual y rework

8. Vertical Slice Inicial

No se debe intentar levantar todo Dofus de una vez. El primer slice jugable recomendado es:

  • 1 servidor visible y funcional
  • razas cargadas
  • creacion de personaje
  • lista de personajes persistida
  • spawn en Incarnam
  • 1 mapa funcional de inicio
  • entrada al mundo
  • movimiento basico

9. Roadmap Por Fases

Fase Objetivo Resultado Esperado
Fase 1 Conexion base Login, seleccion de servidor y ticket resueltos
Fase 2 Personajes CharactersList real y creacion de personaje
Fase 3 Entrada al mundo Spawn en Incarnam y entrada al mapa
Fase 4 Movimiento Desplazamiento valido y persistencia de posicion
Fase 5 Contenido base NPCs, dialogos, inventario y hechizos iniciales
Fase 6 Combate Turnos, hechizos y danos minimos

10. Siguiente Sprint Recomendado

  1. Crear entidad GameServer persistida.
  2. Crear entidades Character, CharacterPosition y CharacterStats.
  3. Crear catalogo Breeds.
  4. Implementar CharactersList real.
  5. Implementar CharacterCreationRequest.
  6. Dejar un spawn fijo en Incarnam.

11. Definition Of Done Del Siguiente Sprint

  • El cliente lista personajes desde DB.
  • Se puede crear personaje sin datos hardcodeados.
  • El personaje queda persistido correctamente.
  • Al seleccionar personaje se asigna un mapa inicial.
  • El flujo auth-to-game no desconecta al cliente.

12. Decision Final

La ruta profesional para este emulador es:

  • DB propia desde cero para runtime
  • importadores para static data
  • primer slice centrado en personajes e Incarnam

Esa combinacion te da control tecnico, mejor mantenimiento y una base sana para seguir creciendo.
